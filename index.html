PyCode Conference 2020

PyCode Conference is a conference devoted to the Python language. The organizers are people from the language community.

The aim of the conference is to raise qualifications of participants and highlight the latest trends in Python. It is also important for us to spread good programming practices.

The conference is addressed to all Python programmers. It opens up to the community at every level of experience which means it is intended for advanced specialists as well as for beginners, including students.

MQTT Pub/Sub: Not just for IoT

This is the basic introductory talk on MQTT. MQTT is a pub/sub protocol for interprocess communication. One process publishes information to a topic, other processes can subscribe to that topic. There is a tree of topics, so it is ideal for managing complexity. It also supports multiple Quality Of Service Levels all the way up to exactly once guaranteed delivery. MQTT is an industry standard, which runs on every platform imagine-able. By the end of this talk you should know enough to decide if MQTT is right for your projects.

ZODB - a mature object-graph database

The ZODB is a mature object-graph database written in Python and optimized in C. Just subclass off of class Persistent Object, and Persistent Container, and your objects, graphs and applications become persistent. Persistent Python, supported by the ZODB allows you to make any Python data structure persistent.

Neo4J and ArrangoDB Graph are the two best known graph databases. But they give you a graph of dictionaries. The developer is responsible for reading and writing. In contrast ZODB gives you a graph of persistent Python objects, sending messages to each other and supporting duck typing. Reading, writing, and caching are all done transparently to the developer. Much easier and faster.

If you are a Python developer, you will find much greater productivity using an object-graph database written in Python, than a graph database written in Java or C. And much much easier to make any desired changes to the database.

The software world is driven by the needs of big data and big corporations. Sure the ZODB with RelStorage can store as much as PostgreSQL, but ZODB is perfect for small data, small companies, and small distributed applications.
